# Audit Methodology

## Objective

The objective of this assessment was to evaluate SecureShop SA's compliance posture against PCI-DSS v4.0 and the Protection of Personal Information Act (POPIA).

## Audit Approach

The assessment followed a risk-based audit methodology consisting of:

1. Planning and Scoping
2. Documentation Review
3. Technical Control Assessment
4. Evidence Collection
5. Control Validation
6. Gap Analysis
7. Risk Assessment
8. Reporting

## Assessment Activities

### Documentation Review

The following documentation was reviewed:

* Information Security Policy
* Access Control Policy
* Password Policy
* Incident Response Plan
* Data Classification Standard
* Privacy Notice
* User Access Reviews

### Technical Assessment

The following technologies were assessed:

* Microsoft Entra ID
* Azure SQL Database
* Azure Key Vault
* Microsoft Sentinel
* Azure Storage Account
* Azure App Services

### Evidence Collection

Evidence was obtained through:

* System screenshots
* Configuration exports
* Administrative interviews
* Policy review
* Log analysis

### Risk Rating Methodology

Likelihood and Impact ratings were assigned using a qualitative model:

* Low
* Medium
* High

Overall Risk Rating:

* Low
* Medium
* High
* Critical

### Compliance Assessment

Controls were assessed as:

* Compliant
* Partially Compliant
* Non-Compliant

### Frameworks Evaluated

* PCI-DSS v4.0
* POPIA

### Limitations

This assessment was based on evidence available during the audit period and did not include penetration testing or forensic validation.
